# Journal Update Workflow

This workflow updates `JOURNAL.md` with a new entry documenting recent changes to the Clip app.

## Steps

1. **Scan for changes**

   - Check git status or file timestamps to identify new/modified files
   - Read key files to understand what changed

2. **Detect the vibe**

   - Bug fix? Feature addition? Refactor? Experimentation?
   - Was it AI-driven or human-driven?
   - What was the energy? (flow state, debugging grind, collaborative iteration)

3. **Write the entry**

   - Add a new section with timestamp
   - Use the narrative voice (conversational, reflective, philosophical)
   - Include:
     - What changed (files, features)
     - Why it matters (connects to PRD goals)
     - The vibe (the feeling/energy of the session)
   - Use code snippets, file references, and emojis

4. **Update the checklist**

   - Mark completed items in "What's Next?" section
   - Add new items if scope expanded

5. **Preserve the philosophy**
   - Keep the meta-commentary about human-AI collaboration
   - Celebrate mistakes as learning opportunities
   - Emphasize iteration over perfection

## Example Entry Format

```markdown
## YYYY-MM-DD - [Session Theme]

### Session N: [Descriptive Title] (HH:MM - HH:MM)

**Human:** [What the human did/said]

**AI:** [What the AI did/observed]

[Narrative about what happened]

**[Key File/Feature]**

- Detail 1
- Detail 2

**The Vibe:** [One-line summary of the energy/feeling]
```

## Notes

- Keep entries concise but evocative
- Focus on _why_ and _how_, not just _what_
- Celebrate both successes and mistakes
- Make it fun to read!
